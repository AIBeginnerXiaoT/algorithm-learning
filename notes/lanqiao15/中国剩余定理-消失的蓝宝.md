# 中国剩余定理（CRT）入门：从「消失的蓝宝」到模板总结

## 1. 题目描述

题目给出两个条件，要求满足条件的最小正整数 `N`：

1. `N + 20250412` 能被 `20240413` 整除。
2. `N + 20240413` 能被 `20250412` 整除。

这类“一个数同时满足多个取模条件”的题目，通常就要想到 **中国剩余定理（Chinese Remainder Theorem, CRT）**。

---

## 2. 先把题目翻译成同余方程组

由整除关系可以得到：

```text
N + 20250412 ≡ 0 (mod 20240413)
N + 20240413 ≡ 0 (mod 20250412)
```

移项后变成：

```text
N ≡ -20250412 (mod 20240413)
N ≡ -20240413 (mod 20250412)
```

为了后面更直观，我们把余数化成非负数：

```text
N ≡ 20230414 (mod 20240413)
N ≡ 9999     (mod 20250412)
```

因为：

```text
-20250412 mod 20240413 = 20230414
-20240413 mod 20250412 = 9999
```

所以本题最终就是求解：

```text
N ≡ 20230414 (mod 20240413)
N ≡ 9999     (mod 20250412)
```

---

## 3. 中国剩余定理回顾

对于方程组：

```text
x ≡ a1 (mod m1)
x ≡ a2 (mod m2)
...
x ≡ ak (mod mk)
```

如果 `m1, m2, ..., mk` 两两互质，那么一定存在唯一解，模 `M = m1 * m2 * ... * mk` 意义下唯一。

两组同余时，标准 CRT 写法是：

```text
M = m1 * m2
M1 = M / m1
M2 = M / m2
```

再求：

```text
y1 = M1 在模 m1 下的逆元
y2 = M2 在模 m2 下的逆元
```

那么答案就是：

```text
x ≡ a1 * M1 * y1 + a2 * M2 * y2 (mod M)
```

### 一个非常重要的前提

**标准 CRT 要求模数互质。**

先检查一下本题：

```text
gcd(20240413, 20250412) = 1
```

所以这题可以直接套标准 CRT。

---

## 4. 把公式套到这道题

令：

```text
a1 = 20230414, m1 = 20240413
a2 = 9999,     m2 = 20250412
```

则：

```text
M  = 20240413 * 20250412
M1 = M / 20240413 = 20250412
M2 = M / 20250412 = 20240413
```

接下来要求：

```text
y1 = 20250412 在模 20240413 下的逆元
y2 = 20240413 在模 20250412 下的逆元
```

然后代入：

```text
N ≡ 20230414 * M1 * y1 + 9999 * M2 * y2 (mod M)
```

最终可以得到最小正整数解：

```text
N = 409876661809331
```

检验一下：

```text
409876661809331 + 20250412  可以被 20240413 整除
409876661809331 + 20240413  可以被 20250412 整除
```

---

## 5. 本题 Java 代码

下面这版代码和你的思路一致，但我稍微整理了一下，写成了更适合以后复习的版本：

```java
import java.math.BigInteger;

public class Main {
    public static void main(String[] args) {
        BigInteger m1 = BigInteger.valueOf(20240413L);
        BigInteger m2 = BigInteger.valueOf(20250412L);

        // 把负余数转成非负余数，阅读起来更直观
        BigInteger a1 = BigInteger.valueOf(-20250412L).mod(m1);
        BigInteger a2 = BigInteger.valueOf(-20240413L).mod(m2);

        BigInteger M = m1.multiply(m2);   // 本题 gcd=1，所以这里就是总模数
        BigInteger M1 = M.divide(m1);
        BigInteger M2 = M.divide(m2);

        // 求逆元：M1 * y1 ≡ 1 (mod m1), M2 * y2 ≡ 1 (mod m2)
        BigInteger y1 = M1.modInverse(m1);
        BigInteger y2 = M2.modInverse(m2);

        BigInteger ans = a1.multiply(M1).multiply(y1)
                .add(a2.multiply(M2).multiply(y2))
                .mod(M);

        System.out.println(ans); // 409876661809331
    }
}
```

---

## 6. 代码里每一步在做什么

### `a1` 和 `a2`

表示两个同余方程里的余数：

```text
N ≡ a1 (mod m1)
N ≡ a2 (mod m2)
```

### `M`

表示总模数。  
本题中模数互质，所以：

```text
M = m1 * m2
```

### `M1` 和 `M2`

它们分别是“去掉当前模数之后剩下的部分”：

```text
M1 = M / m1
M2 = M / m2
```

这样做的目的是让每一项在另一个模数下自动变成 `0`。

### `y1` 和 `y2`

它们是逆元，用来把：

```text
M1 mod m1
M2 mod m2
```

强行调成 `1`，这样才能精确拼出对应的余数。

### `ans`

就是 CRT 的拼接公式：

```text
ans = a1 * M1 * y1 + a2 * M2 * y2
```

最后再对 `M` 取模，拿到最小非负解。

## 7. 拓展中国剩余定理（exCRT）是什么

上面讲的是 **标准 CRT**，它要求所有模数两两互质。  
但很多题里模数并不互质，这时候就要用 **拓展中国剩余定理（Extended CRT, exCRT）**。

最常见的两式合并场景是：

```text
x ≡ a1 (mod m1)
x ≡ a2 (mod m2)
```

### exCRT 什么时候有解

先计算：

```text
g = gcd(m1, m2)
```

只有当：

```text
a2 - a1 能被 g 整除
```

也就是：

```text
(a2 - a1) % g = 0
```

方程组才有解。

如果这个条件不满足，那说明两个同余条件彼此冲突，**无解**。

### exCRT 怎么合并

由：

```text
x = a1 + m1 * k
```

代入第二个式子：

```text
m1 * k ≡ a2 - a1 (mod m2)
```

设 `g = gcd(m1, m2)`，两边同时除以 `g`：

```text
(m1 / g) * k ≡ (a2 - a1) / g (mod m2 / g)
```

这时：

```text
gcd(m1 / g, m2 / g) = 1
```

就可以求逆元了，于是：

```text
k ≡ ((a2 - a1) / g) * inv(m1 / g, m2 / g) (mod m2 / g)
```

再带回去：

```text
x = a1 + m1 * k
```

最后可以把两个方程合并成一个新的同余式：

```text
x ≡ newA (mod lcm(m1, m2))
```

其中：

```text
lcm(m1, m2) = m1 / g * m2
```

这就是 exCRT 的核心思想：  
**每次把两个同余方程合并成一个，再继续和下一个方程合并。**

### exCRT 和 CRT 的区别

- CRT：模数互质，直接套总公式。
- exCRT：模数不一定互质，需要先判定是否有解，再逐步合并。

### 一句复习版总结

以后遇到同余方程组，先问自己两件事：

- 模数是不是互质？
- 如果不互质，差值能不能被 gcd 整除？

如果互质，用 CRT。  
如果不互质，就考虑 exCRT。

## 8. 复习时要记住的几个点

### 1. 遇到“同时满足多个取模条件”，优先想到 CRT

常见题型关键词：

- `x % m1 = a1`
- `x % m2 = a2`
- “同时满足”
- “最小正整数解”

### 2. 先看模数是否互质

- 互质：直接用标准 CRT。
- 不互质：要判断是否有解，通常要用 **扩展中国剩余定理**。

### 3. 余数最好统一成非负

像本题这种负余数：

```text
N ≡ -20250412 (mod 20240413)
```

实际写代码时更推荐改成：

```text
N ≡ 20230414 (mod 20240413)
```

代码更清楚，也更不容易写错。

### 4. 注意数据范围

本题答案已经到了 `4e14` 级别，`int` 肯定装不下，`long` 勉强可以，写得稳一点直接用 `BigInteger` 更省心。

## 9. 可以直接背下来的 CRT 模板

如果以后做的是“模数互质”的标准 CRT 题，可以直接套这个思路：

```java
import java.math.BigInteger;

public class CRTTemplate {
    public static BigInteger crt(BigInteger[] a, BigInteger[] m) {
        BigInteger M = BigInteger.ONE;
        for (BigInteger mod : m) {
            M = M.multiply(mod);
        }

        BigInteger ans = BigInteger.ZERO;
        for (int i = 0; i < a.length; i++) {
            BigInteger Mi = M.divide(m[i]);
            BigInteger yi = Mi.modInverse(m[i]);
            ans = ans.add(a[i].multiply(Mi).multiply(yi));
        }

        return ans.mod(M);
    }
}
```

使用前提只有一句话：

**所有模数必须两两互质。**
