# 异或经典题：找到两个出现奇数次的数

这是一道位运算经典题。

题目模型：

```text
数组中有两种数出现了奇数次，其他所有数都出现了偶数次。
要求找出这两个出现奇数次的数。
```

核心工具是：

```text
异或 ^
```

---

## 1. 完整代码

下面是根据你给的代码整理后的版本：

```java
public class DoubleOddOccurrenceNumbers {

    public static int[] singleNumber(int[] nums) {
        int eor1 = 0;

        for (int num : nums) {
            // nums 中有两种数 a、b 出现了奇数次，其他数都出现了偶数次
            eor1 ^= num;
        }

        // eor1 = a ^ b
        // Brian Kernighan 算法：提取二进制里最右侧的 1
        int rightOne = eor1 & (-eor1);

        int eor2 = 0;

        for (int num : nums) {
            if ((num & rightOne) == 0) {
                eor2 ^= num;
            }
        }

        return new int[]{eor2, eor1 ^ eor2};
    }
}
```

---

## 2. 异或的基本性质

异或有几个非常关键的性质：

```text
x ^ x = 0
x ^ 0 = x
异或满足交换律和结合律
```

所以如果一个数出现偶数次，它们全部异或起来会抵消成 `0`。

例如：

```text
7 ^ 7 = 0
3 ^ 3 ^ 3 ^ 3 = 0
```

这也是这类题能用异或解决的根本原因。

---

## 3. 第一步：全部异或

假设数组里只有两个数 `a`、`b` 出现了奇数次，其他数都出现了偶数次。

把所有数异或起来：

```java
int eor1 = 0;
for (int num : nums) {
    eor1 ^= num;
}
```

偶数次的数会全部抵消。

最后剩下：

```text
eor1 = a ^ b
```

因为 `a` 和 `b` 是两种不同的数，所以：

```text
a ^ b != 0
```

这意味着 `eor1` 的二进制里至少有一位是 `1`。

---

## 4. 第二步：提取最右侧的 1

代码：

```java
int rightOne = eor1 & (-eor1);
```

作用：

```text
提取 eor1 二进制中最右侧的 1。
```

例如：

```text
eor1     = 10110000
-eor1    = 01010000
rightOne = 00010000
```

`rightOne` 只保留最右侧的那个 `1`，其他位都是 `0`。

这个技巧常被归到 Brian Kernighan 位运算技巧里。

---

## 5. 为什么这一步能区分 a 和 b

因为：

```text
eor1 = a ^ b
```

如果 `eor1` 某一位是 `1`，说明 `a` 和 `b` 在这一位上不同。

也就是说：

```text
一个在这一位是 0
另一个在这一位是 1
```

`rightOne` 选出来的就是这样一个位置。

所以可以按照这一位把所有数分成两组：

```text
这一位是 0 的一组
这一位是 1 的一组
```

`a` 和 `b` 一定会被分到不同组里。

---

## 6. 第三步：分组异或

代码里选择了其中一组：

```java
int eor2 = 0;

for (int num : nums) {
    if ((num & rightOne) == 0) {
        eor2 ^= num;
    }
}
```

在这一组里：

- 出现偶数次的数仍然会互相抵消
- `a` 和 `b` 只有其中一个会留在这一组

所以最后：

```text
eor2 = a 或 b
```

另一个数可以用：

```java
eor1 ^ eor2
```

得到。

最终返回：

```java
return new int[]{eor2, eor1 ^ eor2};
```

---

## 7. 举例说明

假设数组是：

```text
[4, 4, 6, 6, 7, 9]
```

其中：

```text
7 出现 1 次
9 出现 1 次
4、6 都出现 2 次
```

第一步全部异或：

```text
eor1 = 4 ^ 4 ^ 6 ^ 6 ^ 7 ^ 9
     = 7 ^ 9
```

二进制：

```text
7 = 0111
9 = 1001
7 ^ 9 = 1110
```

提取最右侧的 `1`：

```text
rightOne = 0010
```

按照这一位分组：

```text
7  = 0111，这一位是 1
9  = 1001，这一位是 0
```

所以 `7` 和 `9` 会被分到不同组。  
每一组内部再异或，就能分别得到这两个数。

---

## 8. 复杂度

```text
时间复杂度：O(n)
空间复杂度：O(1)
```

只需要遍历数组两次，并且只使用几个变量。

---

## 9. 易错点

### 1. `rightOne` 不是随便取一位

必须取 `eor1` 中为 `1` 的某一位。  
因为只有这样的位才能保证 `a` 和 `b` 在这一位上不同。

```java
int rightOne = eor1 & (-eor1);
```

### 2. 分组条件写成 `== 0` 或 `!= 0` 都可以

这两种写法只是选择不同的组：

```java
(num & rightOne) == 0
(num & rightOne) != 0
```

不管选哪一组，最后都会得到 `a` 或 `b` 中的一个。

### 3. 返回顺序不固定

```java
return new int[]{eor2, eor1 ^ eor2};
```

返回的两个数顺序不一定和题目样例一致。  
如果题目要求升序，可以再排序：

```java
int a = eor2;
int b = eor1 ^ eor2;
return a < b ? new int[]{a, b} : new int[]{b, a};
```

### 4. 前提必须是两个数出现奇数次

这份代码适用的前提是：

```text
刚好有两种数出现奇数次，其他数出现偶数次。
```

如果只有一个数出现奇数次，或者有三个以上数出现奇数次，就不是这个模板。

---

## 10. 复习口诀

```text
偶数次异或会消掉。
全体异或得到 a ^ b。
a ^ b 里找一个为 1 的位。
按这一位分组，a 和 b 必分两边。
组内再异或，得到其中一个，另一个用 eor1 再异或回来。
```
