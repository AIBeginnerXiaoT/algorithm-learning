# 位图 Bitset：用 int 数组压缩布尔状态

`Bitset` 是一种用二进制位表示状态的数据结构。

它适合处理这种问题：

```text
有 n 个编号，从 0 到 n - 1。
每个编号只有两种状态：存在 / 不存在，开 / 关，true / false。
```

普通写法可以用 `boolean[]`，而 `Bitset` 会把多个布尔状态压缩到一个整数里。

在 Java 中，一个 `int` 有 `32` 位，所以：

```text
1 个 int 可以表示 32 个状态。
```

---

## 1. 完整代码

下面是根据你给的代码整理后的版本：

```java
public class Bitset {

    private int[] set;

    public Bitset(int n) {
        // n 个状态，每个 int 可以表示 32 个状态
        // 向上取整：(n + 31) / 32
        set = new int[(n + 31) / 32];
    }

    // 加入 num
    public void add(int num) {
        set[num / 32] |= 1 << (num % 32);
    }

    // 删除 num
    public void remove(int num) {
        set[num / 32] &= ~(1 << (num % 32));
    }

    // 反转 num 的状态
    public void reverse(int num) {
        set[num / 32] ^= 1 << (num % 32);
    }

    // 查询 num 是否存在
    public boolean contains(int num) {
        return ((set[num / 32] >> (num % 32)) & 1) == 1;
    }
}
```

---

## 2. 为什么数组长度是 `(n + 31) / 32`

因为一个 `int` 能表示 `32` 个状态。

如果有 `n` 个状态，需要的 `int` 数量是：

```text
ceil(n / 32)
```

整数写法就是：

```java
(n + 31) / 32
```

例子：

```text
n = 32，需要 1 个 int
n = 33，需要 2 个 int
n = 64，需要 2 个 int
n = 65，需要 3 个 int
```

---

## 3. num 存在哪个 int 里

代码：

```java
set[num / 32]
```

含义：

```text
num / 32 决定使用 set 数组中的哪个 int。
```

例如：

```text
num = 0  到 31，存在 set[0]
num = 32 到 63，存在 set[1]
num = 64 到 95，存在 set[2]
```

---

## 4. num 对应 int 的哪一位

代码：

```java
num % 32
```

含义：

```text
num % 32 决定它在当前 int 里的第几位。
```

例如：

```text
num = 35
num / 32 = 1，所以在 set[1]
num % 32 = 3，所以在 set[1] 的第 3 位
```

要构造这一位对应的掩码：

```java
1 << (num % 32)
```

---

## 5. add：把某一位设置成 1

代码：

```java
set[num / 32] |= 1 << (num % 32);
```

核心操作：

```text
或运算 |
```

`|` 的特点是：

```text
只要有一个是 1，结果就是 1。
```

所以用它可以把目标位设置成 `1`，并且不影响其他位。

示意：

```text
原状态：10010000
掩码：  00000100
结果：  10010100
```

---

## 6. remove：把某一位设置成 0

代码：

```java
set[num / 32] &= ~(1 << (num % 32));
```

先得到目标位掩码：

```java
1 << (num % 32)
```

再取反：

```java
~(1 << (num % 32))
```

这样只有目标位是 `0`，其他位都是 `1`。

再和原状态做与运算：

```text
目标位会被清成 0，其他位保持不变。
```

示意：

```text
原状态：10010100
掩码：  11111011
结果：  10010000
```

---

## 7. reverse：反转某一位

代码：

```java
set[num / 32] ^= 1 << (num % 32);
```

核心操作：

```text
异或 ^
```

`^` 的特点是：

```text
相同为 0，不同为 1。
```

所以和 `1` 异或时，目标位会反转：

```text
0 -> 1
1 -> 0
```

其他位和 `0` 异或，不会变化。

---

## 8. contains：查询某一位是否为 1

代码：

```java
return ((set[num / 32] >> (num % 32)) & 1) == 1;
```

步骤：

1. 找到 `num` 所在的 `int`
2. 把目标位右移到最低位
3. 和 `1` 做与运算，取出最低位

如果结果是 `1`，说明存在。  
如果结果是 `0`，说明不存在。

也可以写成：

```java
return (set[num / 32] & (1 << (num % 32))) != 0;
```

这两种写法等价。

---

## 9. 操作复杂度

每个操作都是常数时间：

```text
add：O(1)
remove：O(1)
reverse：O(1)
contains：O(1)
```

空间复杂度：

```text
O(n / 32)
```

相比 `boolean[]`，空间更省。

---

## 10. 易错点

### 1. 编号范围必须合法

这份代码默认：

```text
0 <= num < n
```

如果传入负数或超过范围的编号，会数组越界。

### 2. `1 << 31` 会变成负数

Java 的 `int` 是有符号整数。  
当移动到第 `31` 位时：

```java
1 << 31
```

结果是一个负数。

但这不影响位运算本身，因为我们只关心二进制位。  
如果想减少理解成本，也可以把底层数组改成 `long[]`。

### 3. 如果 n 很大，可以用 long 做下标计算

一般刷题用 `int` 足够。  
如果数据范围极大，构造数组长度时要注意溢出问题。

---

## 11. 适用场景

`Bitset` 常用于：

- 大量布尔状态记录
- 集合存在性判断
- 状态压缩
- 图算法中的访问标记压缩
- 位运算优化

前提是编号最好能映射成连续整数范围：

```text
0, 1, 2, ..., n - 1
```

## 12. 复习口诀

```text
num / 32 找数组位置。
num % 32 找 int 内部位。
add 用或，remove 用与非，reverse 用异或。
contains 取出目标位判断是否为 1。
```
